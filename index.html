JavaScript disabled or unsupported: Enable to automate setup
Firefox users: if you installed NoScript, enable (allow or trust) JavaScript
  1. Otherwise, open a new tab and type about:config in the location (address) bar
  2. Type javascript.enabled in the Search box
  3. Double-click to set javascript.enabled to true
  4. Reload this page
If Internet Explorer restricted this webpage from running scripts or ActiveX controls, click Allow blocked content
  1. Otherwise, click Tools and select Internet options
  2. Select the Security tab within Internet Options
  3. Click Custom level...
  4. Scroll until you find the "Scripting" section
  5. Under "Active Scripting", click Enable
  6. Click OK
  7. Reload this page
Chrome users: if you installed ScriptSafe or NotScripts, enable (allow or trust) JavaScript
  1. Otherwise, open a new tab
  2. Type chrome://settings/content in the location (address) bar
  3. In the "Javascript" section, select Allow all sites to run JavaScript (recommended)
  4. Click Done
  5. Reload this page

Learn Git with these free resources

Git stores snapshots of a folder over time in a repository, allowing work to go on simultaneously along different time-lines called branches, while offering a way merge those branches together. Collaborators copy (fork) a git repository, record (commit) changes, push their work, then ask the original author(s) to fetch, checkout, and merge in those changes. If that made no sense, peruse the curated list of resources below. All links open in new tabs.

Remotes: Setup remote repositories & collaborators
Project hosts including Github and Gitlab require that you use (and verify) your school email to share your work privately with faculty. Github also requires you to apply for an educational discount. Sharing your public SSH key secures network connections without passwords.
Git keeps a list of remote repositories on your machine, so that you can submit (push) your work, and access collaborators' work (fetch, merge, or pull).
The command git remote -v lists all remote repositories, including origin (your personal repository) as well as upstream (the course repository), upon successful set up.
